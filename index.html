Ce projet consiste a développer un algorithme de recherche au sein d'une liste de recettes de cuisine.
Cet algorithme doit présenter des résultats à l'utilisateur le plus rapidement possible.
Pour optimiser la rapidité, un choix doit alors être réaliser entre les méthodes .Filter/.foreach et la fonction native : boucle FOR, de javascript.
Ainsi, dans les deux propositions javascript autour des recettes : "Les petits plats", l'ensemble de 3 structures itératives autour de la recherche générale des recettes diffèrent.
Dans la version 1 de l'algorithme, les 3 structures ont recours aux méthodes iteratives liées aux tableaux (Array). La premiere structure utilise une fonction JS .Filter tandis que les deux dernières sont des fonctions JS native .Foreach.
Dans la version 2 de l'algorithme, les 3 structures ont recours à la boucle For, native de JS. Une variable d'iteration se charge de controler les tours de la boucle et répète les instructions suivant les conditions.
Les performances de première structure de boucle des deux versions différentes sont testés sur le site jsben.ch.
Ci-dessous, les deux versions de l'algorithme implémenté sont disponibles à la visite.
L'algorigramme est visible, la version pdf de l'investigation où j'y développe le choix que je préconise est téléchargeable.
